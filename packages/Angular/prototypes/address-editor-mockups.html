Address Editor Widget - Design Options

Reusable Angular component for managing addresses linked to any entity via the polymorphic AddressLink table. Inputs: EntityName + RecordID. Handles Address + AddressLink creation/update in a single UX flow.

Option A: Card List - Each address is a full card with type badge and primary indicator
- Primary address gets a green border/background highlight
- Star icon on non-primary cards toggles primary status
- Edit/Add opens an inline form below the cards
- Best when: addresses are important and form has room to show them prominently
- Trade-off: takes the most vertical space

Option B: Compact Table - Rows are compact with icon, one-line address, type badge, and primary badge
- Action buttons appear on hover (edit, star, delete)
- Edit slides down as an inline panel within the table
- Add button lives in the table footer
- Best when: addresses are secondary info, form is already dense
- Trade-off: less visual prominence, address preview is truncated

Option C: Accordion - Collapsed items show type icon, label, and one-line preview
- Expand to see full address details in a label:value grid
- Edit mode replaces the detail grid with form fields inline
- Primary gets green background on the header row
- Best when: many addresses possible, space is at a premium
- Trade-off: requires a click to see full details
