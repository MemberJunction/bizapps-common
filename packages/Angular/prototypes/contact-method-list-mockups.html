Contact Method List Widget - Design Options

Reusable Angular component for managing contact methods (emails, phones, social profiles) for a Person or Organization. Inputs: PersonID or OrganizationID. Icons come from the ContactType.IconClass field.

Option A: Grouped by Type - Contact methods organized under type headings (Email, Phone, Social)
- Each group has its own inline "add" row at the bottom
- Primary star within each type category
- Click value to copy to clipboard, hover for action buttons
- Best when: many contacts of different types, want clear organization
- Trade-off: takes more vertical space, may over-organize small lists

Option B: Flat List with Icons - Single flat list, each row has type-specific colored icon
- Sub-text shows type name + label (e.g., "Work Email", "Mobile Phone")
- Primary badge on far right, actions appear on hover
- Add form slides in at the bottom with type dropdown + value + label
- Best when: simple and scannable, moderate number of contacts
- Trade-off: can get long with many contacts, no grouping hierarchy

Option C: Two-Column Grid - Contact methods as compact cards in a 2-column grid
- Primary indicator as small star in corner
- Hover reveals copy/delete actions
- "Add" tile is part of the grid flow
- Add form expands to full width below the grid
- Best when: embedding in a dense form, want compact visual impact
- Trade-off: truncates long values (email, URLs), less room for labels
