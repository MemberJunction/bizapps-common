Relationship List Widget - Design Options

Reusable Angular component for viewing and managing relationships between People and Organizations. Uses ForwardLabel/ReverseLabel from RelationshipType for natural language display. Groups by Category (PersonToPerson, PersonToOrganization, OrganizationToOrganization).

Option A: Grouped Timeline - Relationships grouped by Category with icon headers
- ForwardLabel creates natural sentences: "is employee of Acme Corp"
- Color-coded left border shows status (green=Active, orange=Inactive, gray=Ended)
- Title (job role) shown in italics, dates with calendar icon
- Clicking target name navigates to that Person/Org record
- Best when: relationships are a key part of the form, want rich context
- Trade-off: takes the most vertical space

Option B: Compact Table - Dense table with columns: Type, Target, Title, Dates, Status
- Category section dividers within the table
- Ended relationships shown at reduced opacity
- Actions appear on row hover
- Best when: many relationships, need to scan quickly, space is tight
- Trade-off: loses the natural language feel, less visual warmth

Option C: Visual Connection Cards - Each relationship shown as a visual "connection" card: From -> Label -> To
- Avatars with initials (Person) or icons (Org) on each side
- Arrow direction shows directionality; bidirectional for symmetric relationships (Spouse)
- ForwardLabel text sits above the arrow connector
- Right side shows status badge, title, and date range
- Add form includes PersonPicker/OrgPicker with typeahead search
- Category auto-fills based on selected RelationshipType
- Best when: want a visually engaging, CRM-like feel
- Trade-off: most horizontal space, cards can feel wide on narrow layouts
